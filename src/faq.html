--- layout: base title: FAQ Symptom: FAQ | Green sections: - name: General faqs: - title: > My device crashed content: >

Symptom: The device was set up successfully and has been used for a while. However, it crashes when playing music and turning it up loud.

Resolution: Make sure the USB-C plug supplies enough power. The device requires 5 V DC, 2 A. Try a different plug.

- title: > Can’t connect to Wi-Fi during onboarding content: >

Symptom: During onboarding of the device, you cannot connect the device to your Wi-Fi.

Resolution: Try the following steps.

  1. On your phone, make sure you have notifications and location enabled for the Home Assistant Companion App.
    • If you are using an Android phone, make sure precise location is enabled.
  2. If you have different SSIDs (Wi-Fi names) for the 2.4 GHz and the 5 GHz Wi-Fi networks, make sure you use the credentials for the 2.4 GHz network.
  3. If you are using a Bluetooth proxy:
    • Make sure the proxy is close to the device and to the Home Assistant server.
    • If it still does not work, try a different Bluetooth proxy.

- name: Wake word faqs: - title: > The device doesn’t react to my wake word, what can I do? content: >

Symptom: The device was set up successfully and has been used for a while. But now, it no longer responds to the wake word.

Resolution: Try a different wake word. The standard options are Okay Nabu, Hey Jarvis, and Hey Mycroft.

- title: > I changed the wake word but it does not apply, what can I do? content: >

Symptom: You changed the Assistant or the wake word or both via the normal configuration settings shown below, but the changes are not applied.

Resolution: use the wizard

  • To change the wake word, follow the steps on Changing the wake word or Assistant.

  • Info: The wake word and Assistant choices are stored on the device and can only be changed via the wizard. The normal configuration settings under Settings > Voice assistant are not written to the device and don’t apply.
